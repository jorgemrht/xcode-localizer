# Xcode 26 Localization Notes

This skill is Xcode 26-only.

It may persist only:

```text
Translations/Localizable.xcstrings
Translations/AppStrings.swift
Translations/reports/*.html
```

`Translations/Localizable.xcstrings` is the localization source of truth. `Translations/AppStrings.swift` is a generated project wrapper over `LocalizedStringResource`, grouped by screen, for example `AppStrings.Login.buttonSignIn`.

Do not persist Xcode's generated `extension LocalizedStringResource` in the app target. Xcode may generate those symbols automatically from the String Catalog, which can cause duplicate declarations. Do not use `NSLocalizedString` for the generated Swift wrapper.

If a previous `Translations/L10n.swift` exists and contains `Auto-generated by xcode-localizer`, replace that legacy output by deleting it after writing `Translations/AppStrings.swift`.

Every localization change must also write:

```text
Translations/reports/latest.html
Translations/reports/<timestamp>-l10n.html
```

Minimum catalog entry:

```json
{
  "sourceLanguage": "en",
  "version": "1.1",
  "strings": {
    "login_button_sign_in": {
      "comment": "Button that starts login on the login screen.",
      "extractionState": "manual",
      "localizations": {
        "en": {
          "stringUnit": {
            "state": "translated",
            "value": "Sign in"
          }
        },
        "es": {
          "stringUnit": {
            "state": "translated",
            "value": "Iniciar sesion"
          }
        }
      }
    }
  }
}
```

Validate with:

```bash
xcrun xcstringstool print Translations/Localizable.xcstrings
xcrun xcstringstool compile Translations/Localizable.xcstrings --output-directory /tmp/xcode-localizer-check --dry-run
```
